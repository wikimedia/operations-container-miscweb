Comment 15 Bawolff (Brian Wolff) 2011-01-20 01:13:45 UTC 
re-opening this.

I think the preference should perhaps be removed, or at least statistics gathered on how popular it is. See discussion on Bug 26716 and Bug 7092

*The external editor feature doesn't need this pref to work, can be replaced by a bookmarklet (and the instructions say that approach may even be more practical)
*Does anyone use it? (Stats would be nice)
*If accidentally selected, totally breaks editing, without any clear indication as to what is going on.

Thus i think we should seriously consider removing it (or even hiding by default, although having a hidden by default pref seems stupid).
Comment 16 Bawolff (Brian Wolff) 2011-01-20 01:29:46 UTC 
^demon was kind enough to get some stats for this. For en wikipedia, 12158 have the external edit preference set, and 10878 have the externaldiff preference set. This includes both active and inactive users.
Comment 17 Bawolff (Brian Wolff) 2011-01-20 02:05:48 UTC 
before i forget, related to this, i committed r80608 which adds a help link to those preferences so people can figure out what they do, and r80610 which adds a comment to the ini file that this option causes people to download explaining whats going on. (However, its mime type is not that of a text file, so users probably won't see the comment, but it might help some people figure out whats going on).
