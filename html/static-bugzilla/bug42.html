Wikimedia has migrated from Bugzilla to Phabricator. Bug reports should be created and updated in Wikimedia Phabricator instead. Please create an account in Phabricator and add your Bugzilla email address to it.
Wikimedia Bugzilla is read-only. If you try to edit or create any bug report in Bugzilla you will be shown an intentional error message.
In order to access the Phabricator task corresponding to a Bugzilla report, just remove "static-" from its URL.
You could still run searches in Bugzilla or access your list of votes but bug reports will obviously not be up-to-date in Bugzilla.

 The search function has generally improved, but it
could improve even more, I think:

searching for &quot;circle of confusion&quot; yields no results.
if there are no results because of &quot;of&quot;, maybe the
search should ignore the word and match results on
remaining terms. then it would have found what &quot;circle
confusion&quot; found, i.e. [[depth of field]]. (people are
used to google, which drops common words automatically)

some useful words have only two letters. e.g. &quot;pi&quot;.

no raw numerical search turns up results. e.g.
&quot;3.1428&quot; or &quot;8 1/2&quot;. I doubt that will be crucial, but
it would be nice to have the option.
