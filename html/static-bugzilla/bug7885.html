Description Dirk Beetstra 2006-11-12 15:33:20 UTC 
In Opera (version 9.01, build 8552, Sun java 1.5, Win32 XP), when a line with Hebrew text is shown in 
the watchlist (maybe these specific characters), it 'breaks' at the end of the Hebrew text, resulting in 
the next line to be formatted wrongly.  In the history of the page, the formatting is OK.  

What I see, begins with a normal line (no triangle, a 'm', the time, "Silicon"), that ends after the 
Hebrew text (after the last character here: צורן), so you don't see the closing ")", the next line 
begins with: 10:24, a tab, the triangle (so these are wrong way around), and then resumes normally with 
the next item "Organic chemistry".

Comment 2 Rotem Liss 2006-11-12 15:52:11 UTC 
As far as I understand, Unicode standard says the <br /> doesn't break the
direction block, though IE didn't hear about that. Therefore, this behavior is
like that also in Gecko (Mozilla, Firefox): it's not a browser bug, but a
MediaWiki bug.

It seems that <br /> should not be used in the enhanced recent changes view, and
<div> should be used instead. Alternatively, a direction mark could be appended
to the end of the line, though I'd recommend against this way.
