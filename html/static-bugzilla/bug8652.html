Comment 9 Brion Vibber 2007-01-26 10:53:08 UTC 
The root problem may be that the djvutoxml program needed to extract the
metadata is missing on several servers.

As a result, the metadata doesn't get properly loaded.

Then, the case where the data is missing is handled poorly at both low and high
levels, leading to the fun crashes:

* An empty metadata field is not handled properly as a miss case, as the object
ends up with serialize(array()) instead of '0' or ''

Additionally, this seems to result in a null metadata field getting saved, which
breaks MySQL 5 strict mode sometimes....

* Empty/null XML metadata is fed to SimpleXML instead of being calmly ignored.
Warnings are suppressed, but there's nothing to handle the exceptions (perhaps
new in later versions of php?) or deal with a missing/broken field if the
exception is found and cleaned up.
